MCAP

MCAP (pronounced "em-cap") is a modular container file format for heterogeneous timestamped data. It is ideal for robotics applications, as it can record multiple streams of structured and unstructured data in a single file.

MCAP works well under various workloads, resource constraints, and durability requirements.

Concepts

Message
The unit of communication between nodes in the pub/sub system.
Channel
A stream of messages which have the same type, or schema. Often corresponds to a connection between a publisher and a subscriber.
Schema
A description of the structure and contents of messages on a channel, e.g. a Protobuf FileDescriptorSet or JSON Schema.

Features

  • Supports heterogeneous data
    • Store messages encoded in a variety of serialization formats (e.g. ROS, Protobuf, JSON Schema, MessagePack, etc.) in a single file
    • Can include metadata and attachments with application-specific data
  • Optimized for high-performance writing
    • Append-only structure
    • Recover partially-written files, even when data recording is interrupted
  • Efficient reading and seeking within files
    • Use indexing to extract data without scanning the entire file, even over remote connections
    • Fast access to summary data
  • Self-contained
    • Embedded message schemas
    • No additional dependencies required for decoding
  • Reliable under various limitations
    • Workload sizes
    • Resource constraints
    • Durability requirements
